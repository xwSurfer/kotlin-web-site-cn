对比Scala

设计Kotlin语言的两大目的:

  • 编译至少和Java一样快
  • 保证表达到位的同时，语言足够简洁

稍作考虑, 如果你对Scala已相当得心应手,你或许不需要再学习Kotlin

Scala有什么Kotlin没有的

  • 隐式转换, 限定性因素, 等等
    • 在Scala中, 由于画面中有太多的隐式转换，有时不使用debugger,会很难弄清code中具体发生了什么
    • 为了做到功能多样性，Kotlin会使用扩展函数.
  • 重写类型成员
  • 路径依赖性类型
  • 宏
  • 存在类型
    • 类型预测 是一种非常特殊的情况
  • 特性初始化的复杂逻辑
    • 参照 类和接口
  • 自定义符号运算
    • 参照 运算符重载
  • 嵌入式 XML
    • 参照 类型安全 Groovy风格 builders

未来Kotlin或许要增加的内容:

  • 结构类型
  • 值类型
  • Yield operator
  • Actors
  • 并行集合

Kotlin有什么Scala没有的

  • 零开销 空安全
    • Scala的“语法和运行时包装”选项
  • 智能转换
  • Kotlin的内联函数助力非局部跳跃
